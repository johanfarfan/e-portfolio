The Web Project Planning and Management Guide

Picture courtesy of pexels.com by rawpixie

What The Client Wants

Either when a work is done for a client or for self interest and benefit, applications are implemented to solve needs.

Exploring in deep those needs is the first step to start a project. After all, if the application is implemented and no solution was delivered the whole project was a waste of time.



Application Features

Listing all the possible features the Web Site will have will help to make sure that the client is going to have all the required functionality.

It does not hurt to explore or suggest to the client if additional modules would be considered useful to their needs.

Website purpose might vary depending on the client's target, Sales, Donations, Volunteering and Services are the most common ones. Yet it is necessary to create an attractive, engaging website so users would be loyal and attract other potential users.

Due Dates

Even if they are flexible, due dates are a vital part of the agreements.

If the client is in a rush to have the website online and there are a lot of features to work on, scheduled modules can be added according to their priority.

Economics of human resources -how many geeks we have in our team- are our heavier factor for due dates, so it's also good to keep them in mind. Don't forget about another running projects you may have.

Teamwork

Teamwork will guarantee less amount of work and have the project ready for the required date.

After having known all the requirements and before starting to work in anything, the next step is to divide programming and design into modules. For long projects sub divide the longest more complex tasks.

Prioritize the development of each module, the smaller your team is, the more important prioritization becomes

Soloing a extensive project? Sometimes even if we are full stack developers doesn't means we can manage it all in a tight period of time. Looking for someone who can take care of a part of the project makes a huge difference.

On the other hand if our experience isn't adequate for one of the features that our client required, Looking for a remote collaborator is not a bad idea. There are several online directories where you can find web programmers with any kind of specialization, they will happily work for the right share according to their availability.





Time Management Software and other Apps for Productivity


Picture courtesy of pexels.com by Lukas

Productivity software is usually overlooked. For small / micro projects some time management software may be too much, but when working in a team for a big project there are several advantages of using them.

Some other related software for IT professionals have become so important that they are not an option anymore, they are just needed.

Here are some examples of useful apps for professionals

Git

Git has become so important that for the most IT related companies it's use is mandatory. It was designed specially for developers so the code can be shared, modified, by several users even if they are using it at the same time. It basically track changes in your code and it's used for version control.

Other Options
  • Github

Harvest

Time tracking apps like Harvest have several uses. Although it's main use seems to be for analytical data, knowing what is taking most of the time, if everyone has been having the same workload and other details of our team's productivity is an amazing managerial tool.

Other Options
  • Monday
  • Tick

Google Docs

Google Docs alternatives are emerging, still it remains as probably the easiest most practical way to share files and edit them using the same free Online software. It's fast and it allows to edit a file by several users at the same time. It just works wonders.

Other Options
  • Office 365
  • Dropbox
  • One Drive



Final Revision


Picture courtesy of pexels.com by Startup Stock Photos

Test the look and feel

The final phase of the project is more than putting it all together, but a few last tests to experience the feel and try how functional is. Preferably, this is an activity for all the group members, individually.

Test every function

Don't wait for the customer to tell you what works and what doesn't. Be ahead of this situation and inform by yourself about the bugs and give estimates on update deliveries to solve any minor issue.

Document it

Internal documentation for complex function shown as help in the webpage makes the users more independent of any support or training. Don't assume everyone is going to be able to use it intuitively. Popping information for events like mistakes in data entry are also a must.

Manuals are not required for web applications, yet, they should implement small icons for help -usually an interrogation sign that doesn't affect the page aesthetically.

Make it accessible

Accessibility is also more than an optional feature to include, it's a cordial gesture for impaired senses users.

Team meeting pro product delivery

An app developed by a small or a big team should have some final words of either each of the team members -tiny development team,- or groups representants -big development team. If everyone feels good about it the final product is ready, if not, retouches might be needed.
